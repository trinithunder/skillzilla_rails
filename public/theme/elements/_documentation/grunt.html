Working with Grunt (optional)

In one word: automation. The less work you have to do when performing repetitive tasks like minification, compilation, unit testing, linting, etc, the easier your job becomes. After you've configured it through a Gruntfile, a task runner can do most of that mundane work for you—and your team—with basically zero effort.

GruntJS Documentation

How to install Grunt?

To install Grunt, you have to install NodeJs. NodeJS will have npm (node packaged modules). To install Grunt, follow this steps:

  1. Run the following command: npm install -g grunt-cli
  2. Go to grunt folder and copy both files (package.json gruntfile.js) into your project root directory (my-project folder for example).
  3. In your command console, Go to your project directory and run npm install.

Don't hesitate to customize Gruntfile.js in your project root directory if needed.

Commands

grunt

This will minify your assets resources like css, js and folder into a folder called dist.

grunt watch

This will automatically minify your assets when you modify a file.

Troubleshooting

If you have a problem when installing dependencies or running Grunt commands:

  1. Delete the /node_modules/ directory generated by npm
  2. Rerun npm install command
